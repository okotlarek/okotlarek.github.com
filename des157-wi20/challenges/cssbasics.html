Inheritance in CSS

Inheritance refers to styles of defined elements carrying over to undefined elements in CSS based on the order they have been written and default styles. For example, if the font color for all type in the body of a page is defined as pink, all headings and paragraphs will be pink. There are some instances, however, where inheritance is overridden. If there is a link inserted into the body text, it will turn blue and have an underline applied to it because of the pre-defined styles inherent to links in HTML and CSS.

Specificity in CSS

Specificity refers to the rank or hierarchy of each style tag that determines which tag is applied to a given element. In some cases, the style that is defined last is the one that is applied. However, there are some tags that can override others regardless of order, like IDs and class attributes. A compound selector is more specific than using multiple selectors and can allow you to change the style of one specific element within a group with a defined style.

ID and Class Attributes

IDs and class attributes are unique identifiers that can be used in CSS to override the default style hierarchy and allow more specificity. An ID is preceded by a hash character (#) and is the strongest specificity tag in CSS, beating even attribute selectors. A class attribute is preceded by a period (.) and beats out element selectors like h1, p, and div.
